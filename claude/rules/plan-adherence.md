# Plan Adherence

- plan 承認後の実装で、合意した要件 (TX境界、並列戦略、アルゴリズム選択、ライブラリ選定等) を勝手に変更しない
- 合意済みなら確認を挟まず完了まで一気にやりきる (コンパイル・テスト通過まで)
- 逆に、方針が未合意の状態で実装に着手しない

**Why:** 合意済み要件の無断変更は信頼を損なう。一方で中途半端な確認 (「ここまでで一旦止めますか?」) は作業の中断になり、合意済みの作業は完走するのが効率的

**How to apply:**
- plan 承認後、要件変更が必要と判断したら **「この方法だと合意と違う形になるが良いか?」とユーザーに確認** してから進める
- 合意済みなら「次のステップに進んでよいですか?」のような確認は挟まず、テスト/コンパイル/lint まで一気に完走
- 方針未合意で実装に着手しない (plan 段階で合意取得 → 実装に入る、の順序を守る)
