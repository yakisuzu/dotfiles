# Local Development Servers

- ローカル開発用サーバー (web server, dev server, docker compose stack 等) を勝手に起動しない
- 「ローカル環境が必要」な場面では、まず該当ポートが listen 中か確認 (`curl` / `lsof -i :<port>` / `nc -z` 等) → 既に起動済みなら追加起動しない
- 起動が必要と判断した場合も、明示確認を取ってから実行する

**Why:** ユーザーが既にサーバーを起動済みであるケースが多く、追加起動するとポート競合や既存プロセスへの予期せぬ影響を生む

**How to apply:**
- ブラウザテスト・E2E実行・curl確認など「ローカル環境前提」な作業の前に、まずポート確認コマンドを発行
- リッスン無しなら「<port> が空いています、`<command>` を実行してよいですか?」と確認
- リッスンありなら起動コマンドは実行せずそのまま作業
